5章 GitHubの応用的な使い方をマスターしよう

5.1 本章の目標
　
・GitHubの応用的な使い方をマスターすること
　・ファイルの差分を表示する方法
　・バージョンにタグを付ける方法
　・ファイルをまとめてダウンロードする方法
　・ローカル環境とGitHubを連携させたファイル管理方法

5.2 ファイルの差分を表示してみよう

　ファイルが一覧表示されたら、「Code」の下にある時計マークをクリックしましょう。
　すると、samurai-terakoyaリポジトリのこれまでのコミット履歴が時系列で表示されます。
　（変更を登録するときに入力したコミットメッセージが表示されている）

　削除された場合は「Load diff」をクリック

5.3 コミットにタグをつけてみよう
　変更記録にわかりやすい目印を付ける機能。リリースしたバージョンの把握に役立つ
　
〇ユーザーの設定
git config --global user.name "あなたのGitHubユーザー名"
git config --global user.name "YashiroNakai"

〇メールアドレスの登録
git config --global user.email "あなたのGitHubメールアドレス"
git config --global user.email "to_yasuhiro_nakai@yahoo.co.jp"

・設定確認
git config --global --list

2章 HTMLの概要を理解しよう

タグと要素の違い
　タグは<h1>と</h1>のことを指します。
　一方で、要素は<h1>今日の天気</h1>のようにタグで囲んだ中身まで含めたものを指します。
　
空要素
　終了タグがなく開始タグだけで利用できる要素　
　
　空要素	役割	語源
　br	文章を改行する	break
　hr	水平線を表示する	horizontal rule
　img	画像を表示する	image

属性
　HTMLの要素に追加の情報を設定するためのもの
　画像を表示するimg要素では、以下のようにsrc属性を記述して表示したい画像のファイル名を指定します。

　・要素の構成
　要素名 属性名　属性値
　　<img src="photo.jpg">
　　　　 属性＝属性名＋属性値

　・属性は要素に追加の情報をせっていできるもの
　・属性名は属性の名称、属性値は属性の具体的な値

　上記の例は属性値はファイル名（"photo.jpg"）
　属性名は属性値を指定したもの
　その属性を使用して要素が構成される。
　要素は複数の属性で構成される。
　属性名=属性値は対（じゃなければ意味がない）

要素の書き方
　<img src="photo.jpg" width="200" height="100">

<!DOCTYPE html>
　DOCTYPE宣言を記述することで、「このファイルはHTMLである」ことをブラウザが認識できるようになります。
